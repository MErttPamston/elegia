# ⏰ Элегия — Умные повторяющиеся заметки

Минималистичное приложение для заметок с напоминаниями, повторяющимися задачами и Telegram-уведомлениями. Работает как один HTML-файл, хранит данные в браузере.

**[→ Открыть приложение](https://your-username.github.io/elegia/)**

---

## ✨ Возможности

- 📝 Заметки с rich-text редактором (жирный, курсив, подчёркивание)
- ⏰ Напоминания с точным временем
- 🔄 Повторяющиеся задачи — ежедневно, еженедельно, ежемесячно, по будням, nth-weekday
- 🏷️ Теги и цветовые метки
- 📅 Виды: Все, Сегодня, Завтра, Календарь, Выполненные
- 📲 Telegram-уведомления (приходят даже при закрытой вкладке — если настроен сервер)
- 🌙 Тёмная / светлая тема
- 💾 Экспорт / импорт JSON
- 🔁 Синхронизация между вкладками через BroadcastChannel
- ☁️ Опциональное облачное хранение настроек через GitHub Gist
- 🍅 Pomodoro-таймер
- 🤖 AI-ассистент (через Anthropic API)

## 🚀 Быстрый старт

### 1. GitHub Pages (рекомендуется)

1. Создайте репозиторий на GitHub
2. Загрузите `index.html`, `sw.js` в корень репозитория
3. Перейдите в **Settings → Pages → Source: main / root**
4. Приложение будет доступно по адресу `https://your-username.github.io/repo-name/`

> ⚠️ Оба файла должны быть в **одной папке** — `sw.js` должен лежать рядом с `index.html`.

### 2. Локально

Просто откройте `index.html` в браузере. Service Worker не будет работать на `file://`, но все остальные функции — да.

## 📲 Настройка Telegram-уведомлений

### Шаг 1 — Создать бота

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`, придумайте имя и username (например `@MyElегияBot`)
3. Скопируйте **токен** вида `123456789:AABBcc...`

### Шаг 2 — Узнать Chat ID

1. Напишите любое сообщение своему боту
2. Откройте в браузере:
   ```
   https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
   ```
3. Найдите `"chat":{"id":XXXXXXX}` — это ваш Chat ID

### Шаг 3 — Подключить в приложении

1. В Элегии откройте раздел **Telegram** (иконка ✈️ в боковом меню)
2. Введите имя аккаунта, токен бота и Chat ID
3. Нажмите **Сохранить**, затем **🧪 Тест**
4. Если всё верно — придёт тестовое сообщение

## 🖥️ Сервер уведомлений (уведомления без браузера)

По умолчанию уведомления приходят только пока браузер открыт. Чтобы получать их **в любое время** — разверните `server.js` на бесплатном хостинге.

### Бесплатный деплой на Render.com

> Render даёт 750 часов/месяц бесплатно — хватает на один всегда-работающий сервис.

**1. Подготовьте репозиторий**

Убедитесь, что в корне репозитория есть `server.js`. Создайте `package.json`:

```json
{
  "name": "elegia-server",
  "version": "1.0.0",
  "main": "server.js",
  "scripts": {
    "start": "node server.js"
  },
  "engines": {
    "node": ">=18"
  }
}
```

**2. Зарегистрируйтесь на Render.com**

Зайдите на [render.com](https://render.com) → **Sign Up** (можно через GitHub).

**3. Создайте Web Service**

- Dashboard → **New → Web Service**
- Подключите ваш GitHub репозиторий
- Настройки:
  | Поле | Значение |
  |------|----------|
  | Name | `elegia-notifications` |
  | Runtime | `Node` |
  | Build Command | *(оставьте пустым)* |
  | Start Command | `node server.js` |
  | Instance Type | **Free** |
- Нажмите **Create Web Service**

**4. Скопируйте URL**

После деплоя (1–2 минуты) получите URL вида:
```
https://elegia-notifications.onrender.com
```

**5. Вставьте URL в приложение**

В Элегии → раздел **Telegram** → блок **Серверные уведомления** → вставьте URL → **Сохранить**.

Теперь уведомления будут приходить в Telegram даже если браузер закрыт.

> **Важно:** бесплатный инстанс на Render засыпает после 15 минут неактивности. Первое уведомление после долгого перерыва может задержаться на ~30 секунд (пока сервер просыпается). Для моментальной доставки — обновите страницу приложения раз в несколько часов, это синхронизирует заметки с сервером.

### Альтернатива: Railway.app

1. [railway.app](https://railway.app) → **New Project → Deploy from GitHub**
2. Выберите репозиторий с `server.js`
3. Railway автоматически определит Node.js и запустит `npm start`
4. Скопируйте URL в настройки Элегии

## ☁️ Синхронизация настроек через GitHub Gist (опционально)

Если вы пользуетесь приложением с нескольких устройств — настройки Telegram можно хранить в GitHub Gist.

1. Откройте [github.com/settings/tokens](https://github.com/settings/tokens)
2. **Generate new token (classic)** → отметьте scope `gist`
3. Скопируйте токен
4. В Элегии → **Telegram** → секция **GitHub Gist** → вставьте токен → **Подключить**

## 🔔 Как работают уведомления

| Режим | Браузер открыт | Браузер закрыт |
|-------|---------------|----------------|
| Только браузер (по умолчанию) | ✅ Push + Telegram | ❌ |
| + Service Worker (`sw.js`) | ✅ Push + Telegram | ⚠️ Только если вкладка была открыта недавно |
| + Сервер (`server.js`) | ✅ Push + Telegram | ✅ Telegram (всегда) |

## 🛠️ Технологии

- Vanilla JS, CSS custom properties — никаких фреймворков
- localStorage — основное хранилище
- IndexedDB — хранилище внутри Service Worker
- BroadcastChannel API — синхронизация между вкладками
- Notifications API + Service Worker — Push-уведомления
- Telegram Bot API — мессенджер-уведомления
- GitHub Gist API — облачное хранение настроек
- Anthropic API — AI-ассистент
